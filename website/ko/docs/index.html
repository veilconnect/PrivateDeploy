빠른 시작

노드를 배포하고 로컬 프록시 포트를 찾습니다.

데스크톱 앱을 설치하고, 제공자를 추가하고, VPS 노드를 만든 뒤 로컬 Mixed, HTTP, SOCKS 엔드포인트를 사용합니다.

Preview 빌드 설치

플랫폼에 맞는 빌드를 다운로드하고 실행 전에 checksum을 검증합니다.

클라우드 자격 증명 추가

인스턴스와 방화벽 관리 범위의 테스트 API key를 사용하세요.

노드 생성

지원 지역과 플랜을 선택합니다. 큰 플랜은 전체 프로토콜 스택을 사용할 수 있습니다.

프로토콜 세부정보 확인

SS, Hysteria2, VLESS-Reality, Trojan 포트와 링크를 확인합니다.

로컬 포트 사용

개요 페이지는 Mixed, HTTP, SOCKS 포트를 바로 보여줍니다.

다른 클라이언트로 가져오기

링크를 sing-box, Clash, v2rayN, Hiddify 또는 호환 클라이언트에 복사합니다.

로컬 엔드포인트 형식

데스크톱 개요 포트 패널에서 노출되는 형식입니다.

Mixed

호스트와 포트로 복사합니다.

127.0.0.1:7890

HTTP

HTTP 프록시 URL을 기대하는 앱에 사용합니다.

http://127.0.0.1:7891

SOCKS

SOCKS5 엔드포인트를 받는 도구에 사용합니다.

socks5://127.0.0.1:7892

제공자 자격 증명 안내

평가에는 전용 테스트 token을 사용하고 범용 개인 token 재사용은 피하세요.

Vultr

인스턴스 생성, 방화벽 관리, 메타데이터 읽기, 테스트 노드 삭제가 가능한 key를 사용합니다.

DigitalOcean

droplet과 SSH key를 만들 수 있는 token을 사용합니다. 복구는 관리 SSH key를 사용합니다.

SSH host

소유하거나 관리 권한이 있는 호스트만 사용하세요. 배포에는 서비스와 방화벽 권한이 필요합니다.

일반적인 실패

대부분의 preview 문제는 다음 범주에 속합니다.

제공자가 요청 거부

token 범위, 계정 잔액, 지역 가용성, quota를 확인하세요.

SSH가 준비되지 않음

cloud-init 완료를 기다리고 방화벽과 SSH 지원을 확인하세요.

로컬 포트 연결 실패

커널, proxy/TUN 상태, 노드 링크 상태를 확인하세요.
