Inicio rápido

Despliega un nodo y encuentra tus puertos proxy locales.

Instala la app de escritorio, agrega un proveedor, crea un VPS y usa el endpoint Mixed, HTTP o SOCKS local.

Instala una build preview

Descarga una build para tu plataforma y verifica el checksum antes de abrirla.

Agrega credenciales cloud

Usa una API key de prueba limitada a instancias y firewall.

Crea un nodo

Elige una región y plan compatibles. Planes mayores admiten la pila completa.

Revisa detalles de protocolo

Inspecciona puertos y enlaces de SS, Hysteria2, VLESS-Reality y Trojan.

Usa puertos locales

La página de resumen muestra Mixed, HTTP y SOCKS directamente.

Importa en otro cliente

Copia enlaces en sing-box, Clash, v2rayN, Hiddify u otro cliente.

Formatos de endpoint local

Estos formatos aparecen en el panel de puertos del resumen de escritorio.

Mixed

Copia como host y puerto.

127.0.0.1:7890

HTTP

Úsalo para apps que esperan una URL de proxy HTTP.

http://127.0.0.1:7891

SOCKS

Úsalo para herramientas que aceptan endpoints SOCKS5.

socks5://127.0.0.1:7892

Guía de credenciales de proveedor

Usa un token de prueba dedicado. No reutilices un token personal general.

Vultr

Usa una clave capaz de crear instancias, administrar firewall, leer metadatos y eliminar nodos de prueba.

DigitalOcean

Usa un token que pueda crear droplets y claves SSH. La recuperación usa una clave SSH administrada.

SSH host

Usa hosts propios o autorizados. El despliegue necesita permisos para servicios y firewall.

Fallos comunes

La mayoría de problemas preview caen en estas categorías.

El proveedor rechazó la solicitud

Revisa alcance del token, saldo, disponibilidad regional y cuota.

SSH nunca quedó listo

Espera cloud-init, confirma firewall y verifica soporte SSH.

El puerto local no conecta

Confirma kernel, estado proxy/TUN y enlaces sanos del nodo.
