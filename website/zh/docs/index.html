快速入门

部署节点并找到本地代理端口。

安装桌面端、添加云厂商、创建 VPS 节点，然后在本地使用 Mixed、HTTP 或 SOCKS 端口。

安装预览版

下载适合平台的安装包，并在启动前校验 checksum。

添加云凭证

使用只包含实例和防火墙管理权限的测试 API key。

创建节点

选择支持的地域和套餐。更高内存套餐可部署完整协议栈。

查看协议详情

检查 SS、Hysteria2、VLESS-Reality 和 Trojan 的端口与链接。

使用本地端口

概览页直接显示 Mixed、HTTP 和 SOCKS 端口。

导入其他客户端

把协议链接复制到 sing-box、Clash、v2rayN、Hiddify 或其他客户端。

本地端点格式

这些格式来自桌面端概览页的端口面板。

Mixed

按主机和端口复制。

127.0.0.1:7890

HTTP

用于需要 HTTP 代理 URL 的应用。

http://127.0.0.1:7891

SOCKS

用于支持 SOCKS5 端点的工具。

socks5://127.0.0.1:7892

云厂商凭证建议

评估时使用专用测试 token，不要复用个人全权限 token。

Vultr

使用可创建实例、管理防火墙、读取元数据并删除测试节点的 key。

DigitalOcean

使用可创建 Droplet 和 SSH key 的 token。恢复流程会使用托管 SSH key。

SSH host

只使用你拥有或被授权管理的主机。部署需要安装服务和防火墙规则的权限。

常见失败

预览版问题大多落在以下几类。

云厂商拒绝请求

检查 token 权限、账户余额、地域可用性和实例配额。

SSH 一直未就绪

等待 cloud-init 完成，确认防火墙访问，并检查镜像支持 SSH。

本地端口无法连接

确认内核运行中，系统代理/TUN 状态正确，并且节点链路健康。
