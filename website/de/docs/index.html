Schnellstart

Knoten bereitstellen und lokale Proxy-Ports finden.

Installieren Sie die Desktop-App, fügen Sie einen Anbieter hinzu, erstellen Sie einen VPS-Knoten und nutzen Sie lokal Mixed, HTTP oder SOCKS.

Preview-Build installieren

Laden Sie den passenden Build herunter und prüfen Sie vor dem Start den Checksum.

Cloud-Zugangsdaten hinzufügen

Nutzen Sie einen Test-API-Key für Instanz- und Firewall-Verwaltung.

Knoten erstellen

Wählen Sie eine unterstützte Region und einen Plan. Größere Pläne unterstützen den vollen Stack.

Protokolldetails prüfen

Prüfen Sie Ports und Links für SS, Hysteria2, VLESS-Reality und Trojan.

Lokale Ports nutzen

Die Übersicht zeigt Mixed-, HTTP- und SOCKS-Ports direkt.

Anderswo importieren

Kopieren Sie Links in sing-box, Clash, v2rayN, Hiddify oder einen anderen Client.

Lokale Endpunktformate

Diese Formate zeigt das Port-Panel der Desktop-Übersicht.

Mixed

Als Host und Port kopieren.

127.0.0.1:7890

HTTP

Für Apps, die eine HTTP-Proxy-URL erwarten.

http://127.0.0.1:7891

SOCKS

Für Tools, die SOCKS5-Endpunkte akzeptieren.

socks5://127.0.0.1:7892

Hinweise zu Anbieter-Zugangsdaten

Nutzen Sie einen dedizierten Test-Token. Verwenden Sie keine universellen persönlichen Tokens.

Vultr

Nutzen Sie einen Key, der Instanzen erstellen, Firewall-Regeln verwalten, Metadaten lesen und Testknoten löschen kann.

DigitalOcean

Nutzen Sie einen Token, der Droplets und SSH-Keys erstellen kann. Recovery nutzt einen verwalteten SSH-Key.

SSH host

Nutzen Sie nur eigene oder autorisierte Hosts. Deploy benötigt Rechte für Dienste und Firewall-Regeln.

Häufige Fehler

Die meisten Preview-Probleme fallen in diese Kategorien.

Anbieter lehnt Anfrage ab

Prüfen Sie Token-Umfang, Kontostand, Region und Quote.

SSH wird nie bereit

Warten Sie auf cloud-init, prüfen Sie Firewall und SSH-Unterstützung.

Lokaler Port verbindet nicht

Prüfen Sie Kernel, Proxy/TUN-Zustand und gesunde Links des Knotens.
