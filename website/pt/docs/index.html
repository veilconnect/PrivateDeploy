Início rápido

Implante um nó e encontre suas portas proxy locais.

Instale o app desktop, adicione um provedor, crie um VPS e use o endpoint Mixed, HTTP ou SOCKS local.

Instale uma build preview

Baixe a build da sua plataforma e verifique o checksum antes de abrir.

Adicione credenciais cloud

Use uma API key de teste limitada a instâncias e firewall.

Crie um nó

Escolha uma região e plano compatíveis. Planos maiores aceitam a pilha completa.

Revise detalhes de protocolo

Veja portas e links de SS, Hysteria2, VLESS-Reality e Trojan.

Use portas locais

A visão geral mostra Mixed, HTTP e SOCKS diretamente.

Importe em outro cliente

Copie links para sing-box, Clash, v2rayN, Hiddify ou outro cliente.

Formatos de endpoint local

Estes formatos aparecem no painel de portas da visão geral desktop.

Mixed

Copie como host e porta.

127.0.0.1:7890

HTTP

Use para apps que esperam uma URL de proxy HTTP.

http://127.0.0.1:7891

SOCKS

Use para ferramentas que aceitam endpoints SOCKS5.

socks5://127.0.0.1:7892

Guia de credenciais do provedor

Use um token de teste dedicado. Evite reutilizar um token pessoal genérico.

Vultr

Use uma chave que possa criar instâncias, gerenciar firewall, ler metadados e excluir nós de teste.

DigitalOcean

Use um token que possa criar droplets e chaves SSH. A recuperação usa uma chave SSH gerenciada.

SSH host

Use hosts seus ou autorizados. O deploy precisa de privilégio para serviços e regras de firewall.

Falhas comuns

A maioria dos problemas preview cai nestas categorias.

Provedor rejeitou a solicitação

Verifique escopo do token, saldo, região e quota.

SSH nunca ficou pronto

Aguarde cloud-init, confirme firewall e suporte SSH.

Porta local não conecta

Confirme kernel, estado proxy/TUN e links saudáveis do nó.
