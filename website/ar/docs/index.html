البدء السريع

انشر عقدة واعرف منافذ البروكسي المحلية.

ثبّت تطبيق سطح المكتب، أضف مزودًا، أنشئ عقدة VPS، ثم استخدم منفذ Mixed أو HTTP أو SOCKS محليًا.

ثبّت إصدارًا تجريبيًا

نزّل build المناسب لمنصتك وتحقق من checksum قبل تشغيله.

أضف بيانات السحابة

استخدم مفتاح API تجريبيًا محدودًا لإدارة المثيلات والجدار الناري.

أنشئ عقدة

اختر منطقة وخطة مدعومتين. الخطط الأكبر تدعم الحزمة الكاملة.

راجع تفاصيل البروتوكول

راجع منافذ وروابط SS و Hysteria2 و VLESS-Reality و Trojan.

استخدم المنافذ المحلية

تعرض النظرة العامة منافذ Mixed و HTTP و SOCKS مباشرة.

استورد في عميل آخر

انسخ الروابط إلى sing-box أو Clash أو v2rayN أو Hiddify أو عميل متوافق.

صيغ endpoint المحلية

هذه الصيغ تظهر في لوحة المنافذ في تطبيق سطح المكتب.

Mixed

انسخ كعنوان host ومنفذ.

127.0.0.1:7890

HTTP

للتطبيقات التي تحتاج URL بروكسي HTTP.

http://127.0.0.1:7891

SOCKS

للأدوات التي تقبل endpoints من نوع SOCKS5.

socks5://127.0.0.1:7892

إرشادات بيانات المزود

استخدم token اختبار مخصصًا. لا تعيد استخدام token شخصي شامل.

Vultr

استخدم مفتاحًا يمكنه إنشاء المثيلات وإدارة الجدار الناري وقراءة metadata وحذف عقد الاختبار.

DigitalOcean

استخدم token يمكنه إنشاء droplets ومفاتيح SSH. الاسترداد يستخدم مفتاح SSH مدارًا.

SSH host

استخدم مضيفين تملكهم أو لديك إذن بإدارتهم. يحتاج النشر صلاحيات للخدمات والجدار الناري.

الأعطال الشائعة

معظم مشاكل preview تقع ضمن هذه الفئات.

المزود رفض الطلب

تحقق من صلاحيات token والرصيد والمنطقة والحصة.

SSH لم يصبح جاهزًا

انتظر cloud-init وتحقق من الجدار الناري ودعم SSH.

المنفذ المحلي لا يتصل

تأكد أن kernel و proxy/TUN وروابط العقدة سليمة.
