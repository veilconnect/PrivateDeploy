Быстрый старт

Разверните узел и найдите локальные proxy-порты.

Установите desktop-приложение, добавьте провайдера, создайте VPS-узел и используйте локальный Mixed, HTTP или SOCKS endpoint.

Установите preview-сборку

Скачайте сборку для вашей платформы и проверьте checksum перед запуском.

Добавьте cloud-учетные данные

Используйте тестовый API key с правами на instance и firewall.

Создайте узел

Выберите поддерживаемый регион и план. Большие планы поддерживают полный stack.

Проверьте протоколы

Проверьте порты и ссылки SS, Hysteria2, VLESS-Reality и Trojan.

Используйте локальные порты

Страница overview показывает Mixed, HTTP и SOCKS прямо.

Импортируйте в другой клиент

Скопируйте links в sing-box, Clash, v2rayN, Hiddify или совместимый клиент.

Форматы локальных endpoints

Эти форматы отображает панель портов desktop overview.

Mixed

Копируется как host и port.

127.0.0.1:7890

HTTP

Для приложений, ожидающих HTTP proxy URL.

http://127.0.0.1:7891

SOCKS

Для инструментов, принимающих SOCKS5 endpoints.

socks5://127.0.0.1:7892

Рекомендации по credentials провайдера

Используйте отдельный test token. Не переиспользуйте универсальный личный token.

Vultr

Используйте key, который может создавать instances, управлять firewall, читать metadata и удалять test nodes.

DigitalOcean

Используйте token, который может создавать droplets и SSH keys. Recovery использует managed SSH key.

SSH host

Используйте только собственные или авторизованные hosts. Deploy требует прав для services и firewall rules.

Типичные сбои

Большинство preview-проблем попадает в эти категории.

Провайдер отклонил запрос

Проверьте scope token, баланс, регион и quota.

SSH не стал готов

Дождитесь cloud-init, проверьте firewall и поддержку SSH.

Локальный порт не подключается

Проверьте kernel, proxy/TUN и здоровые links узла.
