Quick start

Node deploy करें और local proxy ports खोजें।

Desktop app install करें, provider जोड़ें, VPS node बनाएँ, फिर local Mixed, HTTP या SOCKS endpoint उपयोग करें।

Preview build install करें

अपने platform के लिए build डाउनलोड करें और launch से पहले checksum सत्यापित करें।

Cloud credentials जोड़ें

Instance और firewall management तक सीमित test API key उपयोग करें।

Node बनाएँ

Supported region और plan चुनें। बड़े plans full protocol stack चला सकते हैं।

Protocol details देखें

SS, Hysteria2, VLESS-Reality और Trojan ports और links देखें।

Local ports उपयोग करें

Overview page Mixed, HTTP और SOCKS ports सीधे दिखाता है।

दूसरे client में import करें

Links को sing-box, Clash, v2rayN, Hiddify या compatible client में कॉपी करें।

Local endpoint formats

ये formats desktop overview port panel में दिखते हैं।

Mixed

Host और port के रूप में कॉपी करें।

127.0.0.1:7890

HTTP

HTTP proxy URL चाहने वाली apps के लिए।

http://127.0.0.1:7891

SOCKS

SOCKS5 endpoints स्वीकार करने वाले tools के लिए।

socks5://127.0.0.1:7892

Provider credential guidance

Evaluation के लिए dedicated test token उपयोग करें। General personal token reuse न करें।

Vultr

ऐसी key उपयोग करें जो instances बना सके, firewall manage कर सके, metadata पढ़ सके और test nodes delete कर सके।

DigitalOcean

ऐसा token उपयोग करें जो droplets और SSH keys बना सके। Recovery managed SSH key उपयोग करती है।

SSH host

अपने या authorized hosts ही उपयोग करें। Deploy को services और firewall privileges चाहिए।

Common failures

अधिकतर preview issues इन buckets में आते हैं।

Provider ने request reject की

Token scope, account balance, region और quota जाँचें।

SSH ready नहीं हुआ

cloud-init पूरा होने दें और firewall तथा SSH support जाँचें।

Local port connect नहीं करता

Kernel, proxy/TUN state और node links की health confirm करें।
