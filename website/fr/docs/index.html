Démarrage rapide

Déployez un nœud et trouvez vos ports proxy locaux.

Installez l’app desktop, ajoutez un fournisseur, créez un VPS, puis utilisez l’endpoint Mixed, HTTP ou SOCKS local.

Installer une build preview

Téléchargez la build adaptée et vérifiez le checksum avant lancement.

Ajouter des identifiants cloud

Utilisez une clé API de test limitée aux instances et au firewall.

Créer un nœud

Choisissez une région et un plan compatibles. Les plans plus grands acceptent la pile complète.

Vérifier les détails

Inspectez les ports et liens SS, Hysteria2, VLESS-Reality et Trojan.

Utiliser les ports locaux

La vue d’ensemble affiche directement Mixed, HTTP et SOCKS.

Importer ailleurs

Copiez les liens dans sing-box, Clash, v2rayN, Hiddify ou un autre client.

Formats des endpoints locaux

Ces formats sont exposés par le panneau des ports dans l’app desktop.

Mixed

Copier sous forme hôte et port.

127.0.0.1:7890

HTTP

À utiliser avec les apps qui attendent une URL de proxy HTTP.

http://127.0.0.1:7891

SOCKS

À utiliser avec les outils qui acceptent des endpoints SOCKS5.

socks5://127.0.0.1:7892

Conseils sur les identifiants fournisseur

Utilisez un token de test dédié. Ne réutilisez pas un token personnel polyvalent.

Vultr

Utilisez une clé capable de créer des instances, gérer le firewall, lire les métadonnées et supprimer les nœuds de test.

DigitalOcean

Utilisez un token capable de créer droplets et clés SSH. La récupération utilise une clé SSH gérée.

SSH host

Utilisez des hôtes que vous possédez ou êtes autorisé à gérer. Le déploiement nécessite des droits services et firewall.

Échecs courants

La plupart des soucis preview entrent dans ces catégories.

Le fournisseur rejette la requête

Vérifiez le périmètre du token, le solde, la région et le quota.

SSH ne devient jamais prêt

Attendez cloud-init, confirmez le firewall et le support SSH.

Le port local ne connecte pas

Vérifiez le kernel, l’état proxy/TUN et la santé des liens du nœud.
