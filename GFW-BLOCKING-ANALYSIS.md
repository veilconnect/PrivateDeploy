## 🔍 regional reachability 屏蔽模式分析

### 1. **IP选择性屏蔽**

**证据**：
- ✅ 旧洛杉矶IP (144.202.124.223) - **完全正常**
- ❌ 新洛杉矶IP (104.207.152.99) - **ICMP被屏蔽，但端口开放**
- ❌ 新加坡IP (192.0.2.1) - **完全被屏蔽**

**结论**：
- **不是按区域全面屏蔽**，而是针对特定IP地址段
- 新分配的IP更容易在regional reachability黑名单中
- 老IP（2025年8月创建）更安全

---

### 2. **屏蔽类型分析**

#### 类型A：完全屏蔽 (sg节点)
```
- ICMP: ✗ 100% packet loss
- TCP端口: ✗ filtered
- SSH (22): ✗ filtered
- 所有VPN协议: ✗ filtered
```
**特征**：所有流量都被丢弃，典型的regional reachability IP黑名单

#### 类型B：ICMP屏蔽，端口开放 (新洛杉矶节点)
```
- ICMP: ✗ 100% packet loss
- TCP端口: ✓ OPEN
- Shadowsocks: ✓ OPEN (46324)
- VLESS: ✓ OPEN (46326)
```
**特征**：
- regional reachability可能屏蔽了ICMP（ping）协议
- 但TCP/UDP端口仍然可以访问
- **VPN可能仍然可用**！

#### 类型C：完全正常 (旧洛杉矶节点)
```
- ICMP: ✓ 0% packet loss
- 所有端口: ✓ 应该可用
```
**特征**：不在regional reachability黑名单中

## 📈 长期建议

### 1. **多节点冗余**

不要只依赖单个节点：
- 保持 2-3 个不同区域的节点
- 如果一个被屏蔽，立即切换到备用节点

### 2. **定期IP轮换**

- 每月检查节点连通性
- 主动删除被屏蔽的节点
- 重新部署获得新IP

### 3. **协议选择**

不同协议的regional reachability检测难度：
- **Trojan + TLS**: 最难检测（伪装成HTTPS流量）
- **VLESS + Reality**: 很难检测（伪装成真实网站）
- **Hysteria2**: 中等难度（基于QUIC）
- **Shadowsocks**: 较容易被检测和屏蔽

**建议**：优先使用 Trojan 或 VLESS-Reality

### 4. **端口选择**

避免常见VPN端口：
- ❌ 避免: 8080, 8388, 443, 1080 等
- ✅ 推荐: 使用PrivateDeploy的随机端口（如 46324）

### 5. **流量特征混淆**

- 使用TLS加密
- 添加流量填充
- 模拟正常HTTPS流量
