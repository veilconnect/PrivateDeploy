Vue 3 响应式系统演示：浅拷贝陷阱

问题说明

左侧使用浅拷贝 { ...proxies.value }，右侧使用深拷贝。

点击"删除代理"按钮，观察哪边的UI会正确更新。

观察更新计数器和日志输出，理解 Vue 响应式系统的工作原理。

❌ 问题示例：浅拷贝

{{ name }}

当前: {{ group.now }}
所有: [{{ group.all.join(', ') }}]
{{ log.message }}

✅ 解决方案：深拷贝

{{ log.message }}

技术分析

  • 浅拷贝问题：const updated = { ...proxies.value } 只复制顶层对象，内部的 group 对象仍然是原始引用。
  • 引用共享：修改 updated.group1.all 时，实际上修改的是原始对象。
  • 响应式失效：当执行 proxies.value = updated 时，Vue 检测到内部对象引用没变，可能不触发更新。
  • 深拷贝解决：完全创建新的对象树，每次赋值都是全新的引用，保证响应式系统正确追踪。
